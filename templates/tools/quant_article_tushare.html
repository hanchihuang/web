Tushare Pro 可下载数据权限

这个页面现在改成 API Key 模式说明页。网页登录拿 Tushare 数据权限的方式已取消。站内 replay 目前以分钟数据以外的接口为主，只有超级管理员生成并发放的 API Key 才能访问 /tushare/ 下的非分钟数据；不同数据类型会按频率和查询参数自适应保留周期。当前销售口径到期日为 2026-12-29。

TARDIS 仍然优先：如果你的核心需求是高频、盘口、期权、现货与交易所历史数据，优先看 TARDIS 数据指南。 Tushare 更适合拿 A 股、新闻、全球指数和跨市场补充数据，不喧宾夺主。

一、只支持 API Key，且当前只开放非分钟数据

当前交付与开通流程
  1. 超级管理员在后台创建或选中访问权限记录。
  2. 超级管理员手工生成并发放站内 replay API Key。
  3. 交付默认分成两类凭证：1 个原生 Tushare token，1 个站内 replay API Key。
  4. 请求站内 replay 时必须携带正确的 X-API-Key，否则无法访问 {{ tushare_example_url }}。
  5. 当前 replay key 范围只包含分钟数据以外的接口，例如 /daily/*、/news*、/status、/symbols、/health、/pro/<api_name>。
示例地址：{{ tushare_example_url }}
鉴权规则
网页登录取消：这个页面不再提供注册框或登录框来开通 Tushare 数据权限。
只有 API Key 生效：接口请求必须带正确的 API Key，且该 Key 对应的权限记录已开通、未过期，才能取数。
分钟数据边界：当前站内 replay 不开放 /tushare/minute/*。A 股分钟和期货分钟如需下载，仍应走原生 Tushare token，并遵守单个 API 每分钟最多约 2 次的原生限频。
其他 Tushare Pro 数据：可通过 /tushare/pro/<api_name> 访问互动易、筹码分布、股票、融资融券、新闻、期货、期权、港股、美股、外汇、全球指数、主连/连续合约、指数、ETF、部分其他类型等非分钟数据；完整目录和示例参数可查 前端目录页，JSON 目录接口仍为 /tushare/pro/catalog。
保存周期自适应：数据会先保存到本地持久化存储，不依赖纯内存缓存；新闻通常保留约 15 分钟，日频类通常保留到北京时间当日 24:00，历史日期查询通常保留 3 到 7 天，基础资料类通常保留 7 天，交易日历类通常保留 30 天。
转接层能力边界：站内 replay 面向实时数据、日频数据和历史批量下载，不包含分钟数据；分钟类速率限制仍以原生 Tushare token 为准。
没有权限就拿不到：Key 错误、Key 未授权、Key 过期，都会直接返回 401/403。

四、补充说明

  • 主连分钟直下：不算直接支持，通常需要先做 `mapping`，再映射到实际合约分钟数据。
  • 当前网站 replay 仍然只开放分钟数据以外的接口；分钟类 API 不在站内 replay 权限范围内，需走原生 Tushare token。
  • 不同数据类型会按频率和查询参数自适应保留周期，不再一刀切使用同一种时效；机器重启后仍可继续读取未过期数据。
  • 交付默认包含 2 类凭证：1 个原生 Tushare token；1 个站内 replay API Key。站内 replay 背后可对接多 token 轮换以缓解部分实时、日频与历史接口的速率限制，但不把分钟数据包装成站内 replay。
  • 当前销售口径到期日为 2026-12-29，页面展示的是站内 replay 能力边界，不等于绕开上游所有原生限频。

五、更适合什么场景

Tushare 更适合做 A 股研究、事件驱动、舆情辅助、港美股补充、全球指数同步和中低频研究所需的数据补全。它不是 TARDIS 的替代品，而是量化数据栈里的另一层。
