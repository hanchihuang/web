鉴权规则
网页登录取消：这个页面不再提供注册框或登录框来开通 Tushare 数据权限。
只有 API Key 生效：接口请求必须带正确的 API Key，且该 Key 对应的权限记录已开通、未过期，才能取数。
分钟数据边界：当前站内 replay 的分钟线入口统一为 /tushare/minute/<symbol>/latest。其中 A 股开放 1MIN、5MIN、15MIN、30MIN、60MIN，国内期货连续/主力符号以及常见具体合约写法开放 1MIN、5MIN、15MIN、30MIN；其他分钟类接口仍不在本次 relay 范围内。
其他 Tushare Pro 数据：可通过 /tushare/pro/<api_name> 访问互动易、筹码分布、股票、融资融券、新闻、期货、期权、港股、美股、外汇、全球指数、主连/连续合约、指数、ETF、部分其他类型等非分钟数据；完整目录和示例参数可查 前端目录页，JSON 目录接口仍为 /tushare/pro/catalog。
周期语义统一：凡是 replay 里的周期类接口，latest 默认都按 latest_completed 解释，也就是“最近一根已闭合周期结果”；如果当前周期尚未闭合，系统不会把进行中的 bar 冒充成完整高开低收。
非周期类 /pro 口径：板块、成分、基本面、映射、新闻列表、交易日历这类 /pro 接口不会强行压成最后一条，而是继续返回完整列表；其中如果语义上仍属于“当前口径”查询，比如当日快照、覆盖最新交易日的日频列表，会额外做 freshness 审查，拿不到符合口径的结果时优先返回 503。
保存与新鲜度自适应：数据会先保存到本地持久化存储，不依赖纯内存缓存；但是否返回给顾客，不只看 TTL，还要看内容时间戳是否达到“当前应有口径”。当前口径不达标时会强制回源，回源后仍不达标就返回 503，而不是吐出几天前或上一周期的旧值。普通新闻接口如 /pro/news 通常保留约 1 小时，/pro/major_news 当前保留 1 天，/pro/express_news 当前保留约 15 分钟，公告接口如 /pro/anns_d 通常按公告日维度保留 3 到 7 天，基础资料类通常保留 7 天，交易日历类通常保留 30 天；非周期类历史查询才会保留更久。
转接层能力边界：站内 replay 面向实时数据、日频数据和历史批量下载；分钟类里目前放开 A 股 1/5/15/30/60MIN latest 与国内期货连续/主力符号 1/5/15/30MIN latest，不等于绕开上游原生权限边界。
没有权限就拿不到：Key 错误、Key 未授权、Key 过期，都会直接返回 401/403。

四、补充说明

  • 主连分钟直下：不算直接支持，通常需要先做 `mapping`，再映射到实际合约分钟数据。
  • 当前网站 replay 在分钟类里支持 A 股 1MIN、5MIN、15MIN、30MIN、60MIN latest，以及国内期货连续/主力符号和常见具体合约写法的 1MIN、5MIN、15MIN、30MIN latest。
  • 板块数据现在已单独纳入 replay 目录分类，可直接按 Tushare 自带接口来拿，例如 /tushare/pro/concept、/tushare/pro/concept_detail、/tushare/pro/index_classify、/tushare/pro/index_member。
  • 公告数据现在也已单独纳入 replay 目录分类，代表接口是 /tushare/pro/anns_d；盘后和晚间公告集中披露时最值得刷新。
  • 主连/连续合约相关目录已经拆出单独分类；如果你需要主连映射或切换到实际合约，可优先看 /tushare/pro/fut_mapping 这一类接口。
  • 部分其他类型数据里已明确包含交易日历、休市安排、基础资料和部分映射接口，不建议再按“可能有，也可能没有”去理解。
  • 上面说的“板块数据”是 Tushare 自带板块口径，不等于同花顺板块；如果你问的是 ths_index、ths_member，那要看原生 token 是否具备单独权限。
  • 不同数据类型会按频率和查询参数自适应保留周期，不再一刀切使用同一种时效；机器重启后仍可继续读取未过期数据。
  • 交付默认包含 2 类凭证：1 个原生 Tushare token；1 个站内 replay API Key。页面这里说的是站内 replay 已包装好的分钟入口，不额外暴露底层抓取实现。
  • 当前销售口径到期日为 2026-12-29，页面展示的是站内 replay 能力边界，不等于绕开上游所有原生限频。
